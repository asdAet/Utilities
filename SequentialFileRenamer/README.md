# SequentialFileRenamer

## Что делает скрипт

Скрипт переименовывает все файлы в одной папке по единому шаблону:

```text
{new_name}_{номер}{расширение}
```

Он сохраняет расширение каждого файла, а имя строит из заданного префикса и порядкового номера.

Пример:

- `document.txt` -> `pass__1.txt`
- `notes.log` -> `pass__2.log`

## Что нужно изменить перед запуском

Открой [main.py](./main.py) и проверь:

- `directory` — папка, где нужно переименовать файлы;
- `new_name` — префикс нового имени.

## Как пользоваться

1. Открой [main.py](./main.py).
2. Укажи путь к папке в `directory`.
3. Укажи префикс в `new_name`.
4. Запусти скрипт:

```powershell
python main.py
```

## Что получится после запуска

Все файлы в указанной папке получат новые имена:

- с сохранением старого расширения;
- с нумерацией от `1`;
- без создания отдельных копий.

В консоли скрипт выведет каждое переименование в формате:

```text
старое_имя -> новое_имя
```

## Как работает внутри

1. Скрипт получает список объектов в папке через `os.listdir()`.
2. Проходит по ним через `enumerate(..., start=1)`.
3. Для каждого объекта определяет расширение.
4. Формирует новое имя по шаблону `new_name_<номер><расширение>`.
5. Проверяет, что объект является файлом.
6. Переименовывает файл через `os.rename()`.
7. Печатает результат в консоль.

## Важные особенности текущей реализации

- обрабатываются только файлы в одной папке;
- подпапки не переименовываются;
- порядок нумерации зависит от порядка, который возвращает `os.listdir()`;
- если `new_name = "pass_"`, получится двойное подчёркивание: `pass__1.txt`;
- резервные копии не создаются;
- при конфликте имени возможно завершение с ошибкой.

## Что получает на вход

- папку с файлами;
- префикс нового имени.

## Что создаёт на выходе

- переименованные файлы в той же папке.

## Когда использовать

- когда нужно быстро привести много файлов к одному шаблону;
- когда нужно сохранить расширения;
- когда нужно пронумеровать набор файлов без ручного переименования.
