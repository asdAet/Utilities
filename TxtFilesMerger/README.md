# TxtFilesMerger

## Что делает скрипт

Скрипт собирает все `.txt` файлы из указанной папки и всех её вложенных каталогов, объединяет их в один итоговый файл, а затем очищает результат:

- убирает пустые строки;
- отфильтровывает строки по списку подстрок;
- удаляет слишком длинные строки;
- удаляет дубли, сохраняя первое вхождение.

Это не просто склейка файлов, а последовательная очистка общего текстового набора.

## Что нужно изменить перед запуском

Открой [main.py](./main.py) и проверь эти переменные:

- `root_source_dir` — откуда брать `.txt` файлы;
- `target_dir` — куда сохранить итоговый файл;
- `output_file_name` — имя итогового файла;
- `filter_strings` — список подстрок, строки с которыми будут исключаться.

## Важный нюанс текущей версии

Сейчас в `filter_strings` указана пустая строка:

```python
filter_strings = [
    "",
]
```

Из-за этого условие фильтрации не пропускает ни одну строку. Перед запуском этот список нужно:

- либо сделать пустым: `filter_strings = []`;
- либо заполнить реальными подстроками, которые нужно исключать.

Файл [filters.txt](./filters.txt) сейчас не используется. Настройка фильтрации идёт только через `filter_strings` внутри кода.

## Как пользоваться

1. Открой [main.py](./main.py).
2. Укажи папку-источник в `root_source_dir`.
3. Укажи папку назначения в `target_dir`.
4. Проверь имя итогового файла в `output_file_name`.
5. Исправь `filter_strings`, чтобы фильтр работал корректно.
6. Запусти скрипт:

```powershell
python main.py
```

## Что получится после запуска

В папке `target_dir` появится итоговый файл, обычно `combined.txt`.

В консоли скрипт покажет:

- сколько файлов обработано;
- сколько строк прошло первичную запись;
- сколько пустых строк удалено;
- сколько длинных строк удалено;
- сколько дублей удалено;
- сколько строк осталось в итоге.

## Как работает внутри

Логика идёт в несколько этапов:

1. Скрипт создаёт целевую папку, если её нет.
2. Через `os.walk()` рекурсивно проходит по `root_source_dir`.
3. Для каждого `.txt` файла читает строки в UTF-8.
4. Во время чтения сразу отсеивает пустые строки и строки по `filter_strings`.
5. Записывает всё в один файл.
6. Затем повторно открывает итоговый файл и удаляет оставшиеся пустые строки.
7. После этого удаляет строки длиннее 100 символов.
8. Затем проходит по файлу ещё раз и убирает дубли через `set()`, сохраняя порядок.
9. В конце печатает статистику.

## Что получает на вход

- папку с `.txt` файлами;
- вложенные подпапки тоже обрабатываются;
- файлы должны читаться в UTF-8.

## Что создаёт на выходе

- один итоговый объединённый файл;
- очищенный набор строк без пустых значений, длинных строк и дублей.

## Особенности этого скрипта

- работает рекурсивно по дереву папок;
- делает несколько проходов по итоговому файлу;
- дубли убираются после остальных этапов очистки;
- порядок первых уникальных строк сохраняется;
- все пути задаются прямо в коде.

## Когда использовать

- когда нужно собрать много текстовых файлов в один;
- когда нужно почистить результат от мусора и повторов;
- когда нужен общий текстовый файл для дальнейшего анализа.
