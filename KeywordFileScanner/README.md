# KeywordFileScanner

## Что делает скрипт

Скрипт ищет заданные слова или фразы в файлах выбранной папки. Он работает в интерактивном режиме:

- спрашивает папку для поиска;
- спрашивает расширения файлов;
- читает список слов из `search.txt`;
- создаёт отчёт с найденными совпадениями.

## Что нужно подготовить перед запуском

В этой папке должен быть файл `search.txt` со словами для поиска. Каждое слово или выражение нужно писать с новой строки.

Пример:

```text
error
warning
login
user_id
```

Дополнительно подготовь папку, в которой будут проверяться файлы.

## Как пользоваться

1. Убедись, что рядом со скриптом есть `search.txt`.
2. Запусти скрипт:

```powershell
python main.py
```

3. Когда скрипт попросит путь, введи папку для поиска.
4. Когда скрипт попросит расширения, введи их через запятую, например:

```text
txt, json, cfg
```

5. Дождись окончания поиска.
6. Открой результат в папке `output`.
7. Если хочешь выполнить ещё один поиск без перезапуска, введи новый набор расширений.
8. Чтобы завершить программу, введи `выход`.

## Что получится после запуска

Скрипт создаёт папку `output` рядом с собой и записывает туда:

- `log.txt` — какие файлы были найдены по выбранным расширениям;
- `result.txt` — какие слова были найдены, в каких файлах, строках и позициях.

## Как работает внутри

Логика такая:

1. Скрипт запрашивает у пользователя путь к папке.
2. Затем запрашивает список расширений.
3. Для каждого расширения через `glob.glob()` ищет подходящие файлы.
4. Список найденных файлов записывает в `output/log.txt`.
5. Читает слова из `search.txt`.
6. Открывает каждый найденный файл в UTF-8.
7. Проверяет каждую строку на наличие каждого слова без учёта регистра.
8. Если есть совпадение, сохраняет:
   - путь к файлу;
   - имя файла;
   - слово;
   - номер строки;
   - позицию в строке.
9. Все результаты записывает в `output/result.txt`.

## Важные особенности текущей реализации

- поиск файлов не рекурсивный, только в одной указанной папке;
- поиск по словам нечувствителен к регистру;
- позиция слова вычисляется через `line.find(word)`, поэтому при различии регистра позиция может отображаться неидеально;
- если `search.txt` отсутствует, поиск по словам фактически будет пустым;
- код ориентирован на файлы, которые читаются как UTF-8.

## Что получает на вход

- путь к папке для поиска;
- список расширений;
- файл `search.txt` со словами.

## Что создаёт на выходе

- папку `output`;
- лог по найденным файлам;
- отчёт по найденным словам и их позициям.

## Когда использовать

- когда нужно быстро проверить папку на набор ключевых слов;
- когда нужен отчёт по нескольким типам файлов;
- когда важно видеть не только совпадение, но и строку/позицию.
